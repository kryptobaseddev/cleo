# Installation Guide

This guide covers installation for both end users and contributors.

## System Requirements

<CardGroup cols={3}>
  <Card title="Node.js" icon="node-js">
    Version 24+ (LTS recommended)
  </Card>
  <Card title="Operating System" icon="desktop">
    Linux, macOS, or WSL2 on Windows

  <Card title="Disk Space" icon="hard-drive">

## For End Users

### Option 1: npm Install (Recommended)

```bash
npm install -g @cleocode/cleo
```

This installs the `cleo` and `ct` commands globally. Verify with:

```bash
cleo version
```

### Option 2: MCP-Only (No Global Install)

If you only need CLEO as an MCP server for your AI tools, no global install is required:

```json
{
  "mcpServers": {
    "cleo": {
      "command": "npx",
      "args": ["-y", "@cleocode/cleo@latest", "mcp"]
    }
  }
}
```

This runs CLEO on-demand via `npx` whenever your AI tool connects.

### Option 3: From Source (Contributors)

```bash
git clone https://github.com/kryptobaseddev/cleo.git
cd cleo
npm install
npm run build
npm link
```

<Note>
`npm link` creates global `cleo` and `ct` symlinks pointing to your local build. Changes require `npm run build` to take effect.
</Note>

## Project Initialization

After installation, initialize CLEO in each project:

```bash
cd /path/to/your/project
cleo init
```

This creates:

| File / Directory | Purpose |
|-----------------|---------|
| `.cleo/tasks.db` | SQLite database (tasks, sessions, audit log) |
| `.cleo/config.json` | Project configuration (human-editable) |
| `.cleo/project-info.json` | Project metadata |
| `.cleo/schemas/` | JSON Schema definitions for validation |
| `.cleo/.gitignore` | Selective git tracking rules |

<Info>
`cleo init` automatically injects task management instructions into `CLAUDE.md`, `AGENTS.md`, and `GEMINI.md` for AI agent integration.
</Info>

### Auto-Detect Project Type

Use `--detect` to auto-detect your project's framework, test runner, and package manager:

```bash
cleo init --detect
```

This additionally creates `.cleo/project-context.json` with detected configuration that helps AI agents generate framework-appropriate code.

## Updating

```bash
npm update -g @cleocode/cleo
```

After updating, run `cleo init --update-docs` in each project to refresh agent documentation injections.

## Troubleshooting

<AccordionGroup>
  <Accordion title="Command Not Found" icon="circle-exclamation">
    If `cleo` is not found after npm install, ensure your npm global bin directory is in your PATH:

    ```bash
    # Find npm global bin directory
    npm config get prefix

    # Add to PATH if needed (in ~/.bashrc or ~/.zshrc)
    export PATH="$(npm config get prefix)/bin:$PATH"
    ```
  </Accordion>

  <Accordion title="Node.js Version Too Old" icon="circle-exclamation">
    CLEO requires Node.js 24+. Check your version:

    ```bash
    node --version
    ```

    Update via [nodejs.org](https://nodejs.org) or a version manager like `nvm`:

    ```bash
    nvm install 24
    nvm use 24
    ```
  </Accordion>

  <Accordion title="Validation Errors After Init" icon="triangle-exclamation">
    ```bash
    # Check file integrity
    cleo validate

    # Attempt automatic fix
    cleo validate --fix
    ```
  </Accordion>

  <Accordion title="Permission Denied" icon="lock">
    If npm install fails with permission errors:

    ```bash
    # Option 1: Fix npm permissions
    npm config set prefix ~/.npm-global
    export PATH="$HOME/.npm-global/bin:$PATH"

    # Option 2: Use npx instead of global install
    npx -y @cleocode/cleo@latest version
    ```
  </Accordion>
</AccordionGroup>

## Uninstallation

```bash
# Remove global installation
npm uninstall -g @cleocode/cleo

# Remove project data (per project, if desired)
rm -rf .cleo/
```

<Warning>
Removing `.cleo/` permanently deletes all task data. Create backups first with `cleo backup`.
</Warning>
