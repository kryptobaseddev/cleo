Extends `cleo doctor db-substrate` (T10307) with a per-database Drizzle
migration coverage cross-check. For each inventory entry whose
`migrationsDir` is non-null, the survey now:

1. Reads `SELECT hash, created_at FROM __drizzle_migrations ORDER BY id`
   from the open snapshot.
2. Reads every migration directory on disk via Drizzle's own
   `readMigrationFiles` (the same SHA-256(migration.sql) algorithm
   `migrate()` uses).
3. Cross-references the two sets in BOTH directions and surfaces:
   - `orphanRows[]` — journal rows whose hash has no matching file
     (a `reconcileJournal` Scenario 2A signal).
   - `missingFiles[]` — on-disk migrations whose hash is not journaled
     (a Scenario 3 signal — drizzle's `migrate()` will pick them up).

The result lands on each per-DB envelope entry as `migrationCoverage`:

```ts
dbs[<role>].migrationCoverage: {
  applied: number;     // count in __drizzle_migrations
  expected: number;    // count of migration directories
  orphanRows: { hash: string; createdAt: number | null }[];
  missingFiles: { name: string; hash: string }[];
} | null
```

`null` means one of three things and is NOT a failure:
- `entry.migrationsDir === null` (derived/reserved role)
- the DB file does not exist
- the DB exists but has no `__drizzle_migrations` table yet
  (pre-bootstrap state)

Healthy DBs report `applied === expected`, both arrays empty.

New contracts in `packages/contracts/src/doctor.ts`:
`DbSubstrateMigrationCoverage`, `DbSubstrateMigrationOrphan`,
`DbSubstrateMigrationMissing`. New core primitives in
`packages/core/src/doctor/db-substrate.ts`:
`resolveInventoryMigrationsFolder`, `computeMigrationCoverage`.
Both delegate to the existing `resolveCorePackageMigrationsFolder`
SSoT (T1177) so workspace-dev, bundled, and global-install layouts
all resolve correctly without hardcoding any paths.

Adds 7 new tests under
`packages/cleo/src/cli/commands/__tests__/doctor-db-substrate.test.ts`
covering the orphan-row, missing-file, both-drifts, null-coverage,
derived-role, and resolver-normalization branches. Total file count
rises from 16 to 23 cases.

Closes T10311. Saga: T10281.
