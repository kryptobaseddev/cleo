# Migration Guide: v0.4.0 to v0.5.0 (Pagination & MVI Schema)

## Summary

LAFS v0.5.0 introduces conditional pagination validation, MVI field selection and expansion mechanisms, a context ledger schema, and discriminated union page types. Pagination fields are now mode-specific rather than universally required.

**Breaking changes:** 2
**New features:** 4 (context ledger schema, `_fields`, `_expand`, `E_MVI_BUDGET_EXCEEDED`)

---

## Breaking Changes

### 1. Pagination fields are now mode-conditional

**What changed:** In v0.4.0, the `page` object required all 6 fields (`mode`, `limit`, `offset`, `nextCursor`, `hasMore`, `total`) regardless of mode. In v0.5.0, only `mode` is universally required. Each mode declares its own required fields:

| Mode     | Required fields                        | Optional fields    |
| -------- | -------------------------------------- | ------------------ |
| `cursor` | `mode`, `nextCursor`, `hasMore`        | `limit`, `total`   |
| `offset` | `mode`, `limit`, `offset`, `hasMore`   | `total`            |
| `none`   | `mode`                                 | (none)             |

**Before (v0.4.0):** All fields required, cross-mode fields set to null/zero.

```json
{
  "page": {
    "mode": "cursor",
    "limit": 0,
    "offset": 0,
    "nextCursor": "eyJpZCI6IjEwIn0=",
    "hasMore": true,
    "total": null
  }
}
```

**After (v0.5.0):** Only mode-relevant fields present.

```json
{
  "page": {
    "mode": "cursor",
    "nextCursor": "eyJpZCI6IjEwIn0=",
    "hasMore": true
  }
}
```

**Migration action:** Remove fields irrelevant to the declared pagination mode. For offset mode, remove `nextCursor`. For cursor mode, remove `offset` (and `limit` if not used). For none mode, remove all fields except `mode`.

### 2. TypeScript `LAFSPage` is now a discriminated union

**What changed:** The `LAFSPage` interface has been replaced by a discriminated union of `LAFSPageCursor`, `LAFSPageOffset`, and `LAFSPageNone`. Code that accesses mode-specific fields must now narrow the type via the `mode` discriminant.

**Before (v0.4.0):**

```typescript
const page: LAFSPage = envelope.page;
console.log(page.nextCursor); // always accessible
console.log(page.offset);     // always accessible
```

**After (v0.5.0):**

```typescript
const page: LAFSPage = envelope.page;
if (page.mode === "cursor") {
  console.log(page.nextCursor); // OK — narrowed to LAFSPageCursor
}
if (page.mode === "offset") {
  console.log(page.offset);     // OK — narrowed to LAFSPageOffset
}
```

**Migration action:** Add type narrowing checks (`page.mode === "cursor"`) before accessing mode-specific fields.

## New Features

### 1. Context Ledger Schema

A new schema `schemas/v1/context-ledger.schema.json` defines the structure for tracking state across request/response cycles. The ledger has 7 required fields: `ledgerId`, `version`, `createdAt`, `updatedAt`, `entries`, `checksum`, `maxEntries`. Entries are append-only.

TypeScript types `ContextLedger` and `ContextLedgerEntry` are exported from the package.

### 2. MVI Field Selection (`_fields`)

Clients can now request a subset of response fields via the `_fields` request parameter (array of strings). The server returns only the requested fields plus MVI-required fields. Unknown fields are silently omitted. See spec Section 9.2.

### 3. Expansion Mechanism (`_expand`)

Clients can request expanded/nested data via the `_expand` request parameter (array of strings). Unknown expansion fields return `E_DISCLOSURE_UNKNOWN_FIELD`. Expansion depth is limited to prevent unbounded recursion. See spec Section 9.3.

### 4. `E_MVI_BUDGET_EXCEEDED` Error Code

New registered error code for when responses exceed declared MVI budgets (token, byte, or item limits). HTTP status 413, gRPC status `RESOURCE_EXHAUSTED`.

---

## Conformance Changes

### New check: `pagination_mode_consistent`

Verifies that pagination fields match the declared mode. Cursor mode envelopes with `offset` or offset mode envelopes with `nextCursor` will fail this check. Applies to Standard and Complete adoption tiers.

---

## Migration Checklist

- [ ] Update pagination objects to include only mode-relevant fields
- [ ] Add type narrowing for `LAFSPage` discriminated union in TypeScript code
- [ ] Review `_meta.mvi` usage — field selection and expansion are now formally specified
- [ ] Register `E_MVI_BUDGET_EXCEEDED` in any custom error handling
- [ ] Run conformance suite to verify `pagination_mode_consistent` passes
