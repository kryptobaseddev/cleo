## What Was Built

### 1. `cleo docs add <ownerId> [file] [--url <url>] [--desc] [--labels] [--attached-by]`

Attaches a local file (reads bytes, computes SHA-256) or a remote URL to any CLEO owner entity. Owner type is auto-detected from the ID prefix:
- `T###` → `task`
- `ses_*` → `session`
- `O-*` → `observation`
- `D-*`/`dec_*` → `decision`, `L-*`/`lrn_*` → `learning`, `P-*`/`pat_*` → `pattern`

Returns `{attachmentId, sha256, refCount, kind, ownerId, ownerType}`.

### 2. `cleo docs list [--task T###] [--session ses_*] [--observation O###]`

Lists all attachments for a given owner. Returns table of `id`, `sha256` (truncated), `kind`, `mime`, `size`, `description`, `labels`, `createdAt`, `refCount`.

### 3. `cleo docs fetch <attachmentId|sha256>`

Retrieves attachment metadata + bytes by ID (`att_*`) or SHA-256 hex. Files ≤ 1 MB are base64-encoded inline (`bytesBase64`); larger files report `path` only.

### 4. `cleo docs remove <attachmentId|sha256> --from <ownerId>`

Removes an attachment ref from an owner. When `refCount` reaches zero, the blob is purged from disk. Returns `{removed, attachmentId, from, refCountAfter, blobPurged}`.

## Architecture Notes

- `DocsHandler` is stateless: all store calls happen inside switch case bodies, after param validation. No DB calls if required params are missing (safe for testing without mocks).
- `DerefResult` discriminated union from `attachment-store.ts` is used correctly: `status === 'not-found' | 'derefd' | 'removed'`.
- MIME detection from file extension covers ~14 common types; falls back to `application/octet-stream`.
- No `cleo llmstxt` surface exposed — all attachment ops are under `cleo docs`.
