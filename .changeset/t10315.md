Ratifies `.cleo/backups/sqlite/` as the SINGLE canonical destination for ALL SQLite backups produced under a project — auto session-end snapshots from `vacuumIntoBackupAll` AND manual snapshots from `cleo backup add` now write to the same directory using a unified `YYYYMMDD-HHmmss` local-time timestamp. The legacy `.cleo/backups/snapshot/` directory remains a read-only fallthrough for one deprecation window: `listSystemBackups` enumerates both directories and tags legacy entries with `legacy: true`, `restoreBackup` searches the legacy directory when no canonical match is found, and a one-time `DeprecationWarning` (code `CLEO_BACKUP_LEGACY_SNAPSHOT_DIR`) fires per process when the legacy directory is consulted. ADR-013 §10 and ADR-068 (amendment history) record the decision; tests cover canonical write path, rotation scoping (no longer touches coexisting vacuum-snapshot files in the same dir), legacy-dir enumeration, legacy-dir restore fallthrough, and the one-time deprecation warning. Closes the T10316 "missing brain snapshots" class of bug by removing the two-directory drift that caused recovery candidates to disagree across the two producers. Saga: T10281; Epic: T10284.
