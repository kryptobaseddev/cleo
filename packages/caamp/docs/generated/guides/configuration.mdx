## Configuration Interfaces

### `McpConfigFormat`

Supported MCP config file formats.

### `ConfigFormat`

Supported configuration file formats.  - `"json"` - Standard JSON - `"jsonc"` - JSON with comments (comment-preserving via jsonc-parser) - `"yaml"` - YAML (via js-yaml) - `"toml"` - TOML (via iarna/toml)

**Example**

```typescript
const format: ConfigFormat = "jsonc";
```

### `DetectionConfig`

Configuration for detecting whether a provider is installed.

| Property      | Type                    | Description                                                    |
| ------------- | ----------------------- | -------------------------------------------------------------- |
| `methods`     | `DetectionMethod[]`     | Detection methods to try, in order.                            |
| `binary`      | `string \| undefined`   | Binary name to look up on PATH (for `"binary"` method).        |
| `directories` | `string[] \| undefined` | Directories to check for existence (for `"directory"` method). |
| `appBundle`   | `string \| undefined`   | macOS .app bundle name (for `"appBundle"` method).             |
| `flatpakId`   | `string \| undefined`   | Flatpak application ID (for `"flatpak"` method).               |

**Example**

```typescript
const config: DetectionConfig = {
  methods: ["binary", "directory"],
  binary: "claude",
  directories: ["~/.config/claude"],
};
```

### `McpServerConfig`

Canonical MCP server configuration.

| Property  | Type                                  | Description                                       |
| --------- | ------------------------------------- | ------------------------------------------------- |
| `type`    | `TransportType \| undefined`          | Transport type (`"stdio"`, `"sse"`, or `"http"`). |
| `url`     | `string \| undefined`                 | URL for remote MCP servers.                       |
| `headers` | `Record<string, string> \| undefined` | HTTP headers for remote MCP servers.              |
| `command` | `string \| undefined`                 | Command to run for stdio MCP servers.             |
| `args`    | `string[] \| undefined`               | Arguments for the stdio command.                  |
| `env`     | `Record<string, string> \| undefined` | Environment variables for the stdio process.      |

**Example**

```typescript
// Remote server
const remote: McpServerConfig = {
  type: "http",
  url: "https://mcp.example.com/sse",
};

// Local stdio server
const local: McpServerConfig = {
  command: "npx",
  args: ["-y", "@modelcontextprotocol/server-filesystem"],
};
```

### `GlobalOptions`

Global CLI options shared across all CAAMP commands.

| Property  | Type                    | Description                                  |
| --------- | ----------------------- | -------------------------------------------- |
| `agent`   | `string[] \| undefined` | Target agent IDs (repeatable).               |
| `global`  | `boolean \| undefined`  | Operate on global config instead of project. |
| `yes`     | `boolean \| undefined`  | Skip confirmation prompts.                   |
| `all`     | `boolean \| undefined`  | Target all detected agents.                  |
| `json`    | `boolean \| undefined`  | Output as JSON.                              |
| `dryRun`  | `boolean \| undefined`  | Preview changes without writing.             |
| `verbose` | `boolean \| undefined`  | Enable debug logging.                        |
| `quiet`   | `boolean \| undefined`  | Suppress non-error output.                   |

**Example**

```typescript
const opts: GlobalOptions = {
  agent: ["claude-code", "cursor"],
  global: true,
  json: true,
};
```
