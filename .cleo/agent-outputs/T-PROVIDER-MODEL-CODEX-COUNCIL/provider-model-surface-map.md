# Provider/Model Surface Map

| surface | package | class | current provider/model source | hardcoded values | dynamic discovery already present | routing/catalog implication |
|---|---|---|---|---|---|---|
| `crates/cant-router/src/router.rs` | `crates/cant-router` | routing policy | Rust tier match arm in `route_with_caps` | `claude-opus-4-6`, `claude-sonnet-4-6`, `claude-haiku-4-5`, `kimi-k2.5`, tier cost caps, tier latency budgets | None | Must consume required `RoutingConfig`; route output should be provider/model structured, not bare strings. |
| `packages/cant/src/composer.ts` | `packages/cant` | routing policy / spawn composition | Local `TIER_MODELS` constant | Same Claude/Kimi tier matrix | None | Must require routing policy input and mirror shared contracts; otherwise TS remains a second matrix. |
| `packages/contracts/src/operations/llm.ts` | `packages/contracts` | invocation contract | `ModelConfig` with `transport`, `model`, `apiKey`, `baseUrl`, `fallback` | `ModelTransport = 'anthropic' | 'openai' | 'gemini'` | None | Best home for shared `ModelRef`/catalog contract, but transport enum is too narrow for catalog-provider identity. |
| `packages/core/src/llm/registry.ts` | `packages/core` | invocation transport | Backend factory from `ModelConfig.transport` plus override clients | Anthropic/OpenAI/Gemini built-ins | Override clients can be created from config | Route validation must prove selected model is invokable by this layer or an explicit bridge. |
| `packages/adapters/src/registry.ts` | `packages/adapters` | agent adapter registry | Bundled provider discovery | `claude-code`, `opencode`, `cursor`, `pi` | Manifest loading for those bundled providers | Not authoritative for model catalogs; agent harness provider != model provider. |
| `packages/caamp/providers/registry.json` | `packages/caamp` | provider manifest | CAAMP registry metadata and capabilities | Many curated provider IDs and capabilities | Static curated manifest | Useful capability graph, not a live model catalog. Must not be overloaded as LLM model source. |
| `packages/adapters/src/providers/openai-sdk/spawn.ts` | `packages/adapters` | SDK adapter / invocation bridge | Vercel AI SDK OpenAI provider | `gpt-4.1`, `gpt-4.1-mini` | None in this file | Separate adapter defaults; should eventually accept routed `ModelRef`/`ModelConfig`. |
| `packages/adapters/src/providers/openai-sdk/handoff.ts` | `packages/adapters` | SDK adapter / worker archetypes | Worker archetype table | `gpt-4.1-mini` | None | Separate hardcoded worker-model policy, not fixed by CANT-only config. |
| `packages/adapters/src/providers/claude-sdk/spawn.ts` | `packages/adapters` | SDK adapter / invocation bridge | Vercel AI SDK Anthropic provider | `claude-sonnet-4-5` default | None in this file | Needs same routed model contract if SDK adapters are to honor dynamic policy. |
| `packages/core/src/memory/llm-backend-resolver.ts` | `packages/core` | catalog lookup + invocation fallback | Ollama probe, OpenAI-compatible Ollama provider, fallback backend order | Ollama priority list and cold-tier fallback model | `GET http://localhost:11434/api/tags` | Existing local catalog seed; should be refactored into shared catalog source later. |
| `packages/core/src/metrics/model-provider-registry.ts` | `packages/core` | catalog/provider lookup | `models.dev/api.json` index | Provider inference prefixes | Network fetch from `models.dev` | Existing remote catalog-adjacent source; useful as supplemental provider/model lookup, not full policy. |
