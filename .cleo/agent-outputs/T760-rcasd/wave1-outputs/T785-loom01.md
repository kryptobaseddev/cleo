## Summary

`cleo orchestrate start <epicId>` now auto-initializes the RCASD lifecycle
pipeline at the `research` stage when the epic has no existing pipeline
(`initialized: false`). The behavior is idempotent: a second call skips the
auto-init silently and returns `autoInitialized: false`.

## Design Notes

- `recordStageProgress` with `status='in_progress'` matches the pattern used by
  `ensureLifecycleContext` internally — it upserts the pipeline row and the stage
  row atomically, so no duplicate-init errors occur.
- The existing `startStage` function in `lifecycle/index.ts` was NOT used because
  it calls `checkGate` first, which can block on prerequisite enforcement in
  `strict` mode. For a fresh (uninitialized) epic, the first stage is always
  `research` which has no prerequisites, but using `recordStageProgress` directly
  is safer and consistent with the engine pattern.
- Guardrail respected: lifecycle module internals were not modified.
- No new CLI flags were added.
