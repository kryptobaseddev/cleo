## Design Notes

- Types match the required shape from T766 spec exactly. The spec listed `AttachmentCommon` as a base interface; this implementation inlines those fields directly on each variant (same as T779's `GateBase` pattern) while the Zod schemas compose via `attachmentCommonSchema.extend({...})`.
- `AttachmentMetadata.sha256` is an unrestricted string (not length-64) to accommodate `url` and `llmtxt-doc` kinds that may have an empty hash.
- `AttachmentRef.ownerType` matches the spec's 6-value enum: `task | observation | session | decision | learning | pattern`.
- CLI surface (`cleo docs`) is NOT implemented here — deferred to T797 per the task directive.
- Storage implementation is NOT included here — deferred to T796.
