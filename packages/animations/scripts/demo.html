3 В· Progress В· Canon Styles

4 В· Sparks В· One-Shot

4 canon flares

Sparks are finite frame sequences. They play once and decay back to empty вҖ” not loops. awaken at first dream / cleo init; sweep on BRAIN integrity sweep complete; cascade on release-shipped / task-complete; weave on playbook stage transitions and CANT directive acceptance. (On this page they auto-replay every ~1.4s so you can see the motion.)

5 В· LAFS Render Gate

AnimateContext

Every primitive in this package routes its output through an AnimateContext. The context is pure data вҖ” no I/O, no timers вҖ” derived from the LAFS FlagResolution plus environment signals. When silent, primitives return no-op handles so callers don't branch on output mode.

SignalSourceEffectReason code
format !== 'human'LAFS flagsDisable animations (machine output)format-json
quiet === trueLAFS flagsDisable animations (script-friendly)quiet
!isTTYprocess.stdout.isTTYDisable animations (piped/redirected)no-tty
NO_COLOR setprocess.env.NO_COLORDisable animations (no-color.org)no-color

Construct it once at command entry, thread it everywhere downstream:

import { resolveOutputFormat } from '@cleocode/lafs';
import { createAnimateContext } from '@cleocode/animations';

const flags = resolveOutputFormat({ humanFlag: true });
const ctx = createAnimateContext({ flagResolution: flags });

if (ctx.enabled) /* render spinner */;
else console.log(`silent: ${ctx.reason}`);
// вҶ’ "enabled" | "format-json" | "quiet" | "no-tty" | "no-color"
6 В· Usage Guide

How to render

Lint rule: any process.stdout.write(...) of a string starting with \r outside @cleocode/animations is a contract violation. Always route through createSpinnerHandle вҖ” it owns the timer, the cursor, the exit handler, and the LAFS gate.

Spinner during async work вҖ” the canonical pattern
import { resolveOutputFormat } from '@cleocode/lafs';
import { createAnimateContext, createSpinnerHandle } from '@cleocode/animations';

const ctx = createAnimateContext({
  flagResolution: resolveOutputFormat({ humanFlag: true }),
});

const spinner = createSpinnerHandle(ctx, 'looming', 'Weaving tasksвҖҰ');
spinner.start();
try {
  await doWork();
  spinner.stop('вң” Tapestry complete.');
} catch (err) {
  spinner.stop();
  throw err;
}
Multi-step pipeline with label updates
const spinner = createSpinnerHandle(ctx, 'weaving', 'Connecting to databaseвҖҰ');
spinner.start();

const db = await connect();
spinner.update(`Running ${migrations.length} migrationsвҖҰ`);

await db.migrate(migrations);
spinner.stop('вң” Database ready.');
Progress bar with a known ratio
import { renderProgressBar } from '@cleocode/animations';

function tick(done, total) {
  const ratio = done / total;
  const bar = renderProgressBar('refinery', ratio, 36);
  // (progress bars are not yet routed through a handle вҖ”
  //  gate manually with ctx.enabled until ProgressBarHandle ships)
  if (ctx.enabled) process.stdout.write(`\r\x1B[2K  ${bar}  ${done}/${total}`);
}
One-shot spark on success
import { sparks, sparkDurationMs } from '@cleocode/animations';

async function playSpark(name) {
  if (!ctx.enabled) return;
  const { frames, interval } = sparks[name];
  for (const f of frames) {
    process.stdout.write(`\r\x1B[2K  ${f}`);
    await new Promise(r => setTimeout(r, interval));
  }
  process.stdout.write('\n');
}

await shipRelease();
await playSpark('cascade'); // total: ~980ms via sparkDurationMs('cascade')
