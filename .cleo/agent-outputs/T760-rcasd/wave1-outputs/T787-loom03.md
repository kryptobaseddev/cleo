## Design Decisions

- **`startedAt: null`**: `getLifecycleStatus` queries the `lifecycle_stages` table but does not expose `started_at` in its return type. Per task constraint "do NOT modify lifecycle module internals", `startedAt` is set to `null` (valid per `string | null` contract). A future task can expose it by extending `getLifecycleStatus`.
- **Dynamic import**: `getLifecycleStatus` is imported via `await import(...)` inside `taskShowWithHistory` to avoid circular-dependency risks and keep startup cost at zero for the common case (`--history` not set).
- **Graceful empty history**: Any error from `getLifecycleStatus` (no pipeline, DB error) is caught and returns `history: []` rather than propagating the error.
