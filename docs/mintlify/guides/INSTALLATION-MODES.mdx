# Installation Modes

CLEO supports three installation modes optimized for different use cases.

## Mode Summary

| Mode | Best For | Install Command | Updates Via |
|------|----------|-----------------|-------------|
| **npm Global** | End users, production | `npm install -g @cleocode/cleo` | `npm update -g @cleocode/cleo` |
| **npx (MCP-Only)** | AI tool integration, no global install | Config snippet only | Automatic (latest) |
| **From Source** | Contributors, testers | `git clone` + `npm link` | `git pull` + `npm run build` |

## npm Global (Recommended)

Install CLEO globally via npm for CLI and MCP access.

### Install

```bash
npm install -g @cleocode/cleo
```

### Requirements

- **Node.js** 24+ (LTS recommended)
- npm 10+ (included with Node.js)

### Update

```bash
npm update -g @cleocode/cleo
```

### What You Get

- `cleo` and `ct` commands available globally
- MCP server via `cleo mcp`
- Auto-installer via `cleo mcp-install`

### Verify

```bash
cleo version    # Shows current CalVer version
cleo validate   # Checks project data integrity
```

## npx MCP-Only (No Global Install)

Run CLEO on-demand as an MCP server without any global installation. Your AI tool fetches the latest version automatically each time it connects.

### Configure

Add this to your AI tool's MCP configuration:

```json
{
  "mcpServers": {
    "cleo": {
      "command": "npx",
      "args": ["-y", "@cleocode/cleo@latest", "mcp"]
    }
  }
}
```

### Config Locations

| Tool | Config File |
|------|------------|
| Claude Code | `.mcp.json` (project root) |
| Claude Desktop | `claude_desktop_config.json` |
| Cursor | `~/.cursor/mcp.json` |
| VS Code | `.vscode/mcp.json` |
| Gemini CLI | `~/.gemini/settings.json` |

### When to Use

- You only need CLEO as an MCP server for AI agents
- You do not need the `cleo` CLI command
- You want zero-maintenance auto-updating

### Limitations

- No `cleo` or `ct` CLI commands available
- Slightly slower startup (npx fetches on each run)
- Still requires `cleo init` to initialize a project (run via `npx -y @cleocode/cleo@latest init`)

## From Source (Contributors)

Clone the repository and link locally for development.

### Install

```bash
git clone https://github.com/kryptobaseddev/cleo.git
cd cleo
npm install
npm run build
npm link
```

### Requirements

- **Node.js** 24+
- **Git**

### Update

```bash
cd /path/to/cleo
git pull
npm install
npm run build
```

`npm link` creates global symlinks, so changes take effect after `npm run build`.

### When to Use

- Contributing to CLEO development
- Testing unreleased features
- Debugging issues against source code

### Verify

```bash
cleo version    # Should show the local build version
which cleo      # Should point to your npm link location
```

## Switching Modes

### npm to Source

```bash
npm uninstall -g @cleocode/cleo
cd /path/to/cleo-repo
npm link
```

### Source to npm

```bash
cd /path/to/cleo-repo
npm unlink
npm install -g @cleocode/cleo
```

### Any Mode to npx

Simply remove the global install and update your MCP config to use the npx snippet above. No `cleo` CLI will be available, but MCP operations work through your AI tool.

## Agent Installation

After installing CLEO by any method, agent definitions are managed automatically:

```bash
cleo init              # Creates .cleo/ and injects agent docs
cleo init --update-docs  # Refreshes agent doc injections only
```

This installs instructions into `CLAUDE.md`, `AGENTS.md`, and `GEMINI.md` for AI agent integration.
