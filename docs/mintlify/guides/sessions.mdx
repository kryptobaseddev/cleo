# Session Management

CLEO sessions provide structure for work continuity across terminal sessions and Claude conversations. This guide covers session protocols and focus management.

## Core Concepts

<CardGroup cols={2}>
  <Card title="Sessions Persist" icon="database">
    Sessions survive terminal and Claude conversation boundaries. Resume where you left off.
  </Card>
  <Card title="Sessions Coexist" icon="layer-group">
    Multiple sessions can be active simultaneously on different scopes. No need to suspend.
  </Card>
  <Card title="One Active Task" icon="bullseye">
    Each session enforces single-task focus within its scope for clarity.
  </Card>
  <Card title="Audit Trail" icon="history">
    Every operation is logged for complete traceability and recovery.
  </Card>
</CardGroup>

## Session Lifecycle

### Starting a Session

<Tabs>
  <Tab title="Basic">
    ```bash
    cleo session start
    ```
  </Tab>
  <Tab title="With Scope">
    ```bash
    # Work on specific epic
    cleo session start --scope epic:T001

    # Work on task group
    cleo session start --scope taskGroup:T005

    # Phase-filtered scope
    cleo session start --scope epicPhase --root T001 --phase testing
    ```
  </Tab>
  <Tab title="Named Session">
    ```bash
    cleo session start \
      --scope epic:T001 \
      --name "Auth Implementation" \
      --agent opus-1 \
      --auto-focus
    ```
  </Tab>
</Tabs>

### During a Session

```bash
# Set your current task
cleo start T001

# Add progress notes

# View current focus
cleo current

# Complete and move on
cleo complete T001
cleo start T002
```

### Ending a Session

<CodeGroup>
```bash End Session
cleo session end --note "Completed auth middleware"
```

```bash Suspend (Pauseable)
cleo session suspend --note "Waiting for API keys"
```

```bash Close (Permanent)
cleo session close --force
# All tasks must be complete, or use --force
```
</CodeGroup>

## Focus Management

CLEO enforces **single-task discipline** - only one task active at a time within a session scope.

### Why Single-Task Focus?

<Info>
Single-task focus prevents context switching overhead and provides clarity on current work. It's the foundation of CLEO's anti-hallucination design.
</Info>

### Focus Commands

```bash
# Set active task
cleo start T001

# View current focus
cleo current

# Clear focus
cleo stop

# Add session-level progress note

# Suggest next action

```

### Focus vs Task Notes

| Command | Purpose | Storage |
|---------|---------|---------|

| `cleo update T001 --notes "..."` | Task-specific notes | Task's notes array (appends) |

## Session States

| State | Meaning | Resumable |
|-------|---------|-----------|
| `active` | Currently working | N/A |
| `suspended` | Paused (explicit) | Yes |
| `ended` | Work complete for now | Yes |
| `closed` | Permanently archived | No |

## Multi-Session Workflows

Multiple sessions can run concurrently on different scopes:

```bash
# Terminal 1: Auth work
cleo session start --scope epic:T001 --name "Auth"

# Terminal 2: UI work (NO CONFLICT - different scope)
cleo session start --scope epic:T050 --name "UI"

# List all active sessions
cleo session list --status active
```

### Scope Types

| Type | Definition | Example |
|------|------------|---------|
| `task` | Single task only | `--scope task:T005` |
| `taskGroup` | Parent + direct children | `--scope taskGroup:T005` |
| `subtree` | Parent + all descendants | `--scope subtree:T001` |
| `epicPhase` | Epic filtered by phase | `--scope epicPhase --root T001 --phase testing` |
| `epic` | Full epic tree | `--scope epic:T001` |

### Session Binding

When you start or resume a session, CLEO binds that terminal to the session:

```bash
# Start session - binds this terminal
cleo session start --scope epic:T001

# All commands now use this session context
cleo current      # Shows THIS session's focus
cleo start T005  # Sets focus within THIS scope

# Switch to different session
cleo session switch <session-id>
```

## Daily Workflow Pattern

<Steps>
  <Step title="Morning: Check State">
    ```bash
    cleo session list
    cleo dash
    cleo current
    ```
  </Step>

  <Step title="Resume or Start">
    ```bash
    # Resume existing session
    cleo session resume <session-id>

    # Or start new session
    cleo session start --scope epic:T001 --auto-focus
    ```
  </Step>

  <Step title="Work: Task Cycle">
    ```bash
    cleo start T001
    # ... do work ...
    cleo update T001 --notes "Completed validation"
    cleo complete T001
    cleo next  # Suggests next task
    cleo start T002
    ```
  </Step>

  <Step title="Evening: Clean Up">
    ```bash
    cleo complete T002
    cleo archive
    cleo session end --note "Completed auth tasks"
    ```
  </Step>
</Steps>

## Resume Patterns

```bash
# Resume by session ID
cleo session resume abc123

# Resume most recent session
cleo session resume --last

# Resume most recent for specific scope
cleo session resume --last --scope epic:T001

# View session details before resuming
cleo session show abc123
```

## Error Recovery

<AccordionGroup>
  <Accordion title="Multiple Active Tasks" icon="triangle-exclamation">

  <Accordion title="Orphaned Session" icon="ghost">

    ```bash
    cleo session end --force
    ```

  <Accordion title="Scope Conflict" icon="code-branch">

    ```bash
    # Check scope availability
    cleo session list --scope T001

    # Use disjoint scopes
    cleo session start --scope epicPhase --root T001 --phase testing
    cleo session start --scope epicPhase --root T001 --phase polish
    ```
  </Accordion>
</AccordionGroup>
