Formalises the cross-database soft-FK contract that SQLite cannot enforce at
the file level. Every Drizzle column whose value points at a row in a
DIFFERENT `.db` file now carries an
`@cross-db <targetRole>.<table>.<column>` JSDoc tag immediately above the
declaration, with prose identifying which accessor or validator enforces the
soft FK at runtime.

### New CI gate ŌĆö `Cross-DB Annotation Lint (T10324)`

- `scripts/lint-cross-db-annotations.mjs` walks every Drizzle schema file
  under `packages/core/src/store/` and `packages/core/src/agents/`,
  identifies cross-DB columns by snake-case name pattern (`task_id`,
  `session_id`, `epic_id`, `agent_id`, `*_agent_id`, `brain_anchor`,
  `project_path`, `*_db_path`, etc.), and fails the PR when ANY such column
  is missing a `@cross-db` tag.
- Intra-DB FKs (columns that chain `.references(() => ŌĆ”)` or live in the
  tasks-DB schema files) are auto-exempt. Per-line opt-out via
  `// cross-db-annotation-ok: <reason>` for cases the heuristic
  mis-classifies (currently `agent_instances.session_id`/`task_id`).
- Three modes: `--strict` (zero tolerance), `--baseline` (capture current
  state), `--check` (CI default ŌĆö fails only on net-add vs baseline).
- Baseline file `scripts/.lint-cross-db-annotations-baseline.json` seeded
  at 0 ŌĆö the initial pass landed under T10324 annotated every cross-DB
  column simultaneously.
- 10 poison tests under `scripts/__tests__/lint-cross-db-annotations.test.mjs`
  cover every classification branch (annotated, un-annotated, intra-DB FK,
  opt-out, intra-DB file allowlist, baseline + check).

### ADR-068 amendment

New subsection "Cross-DB Reference Columns" between the `openCleoDb`
chokepoint and the CI Gate section. Defines the tag grammar
(`@cross-db <targetRole>.<table>.<column> ŌĆö <direction> <accessor note>`),
enumerates which column-name patterns count as cross-DB, lists the
intra-DB exemption rules, and documents the new CI gate. Amendment
history entry added; status remains `accepted`.
