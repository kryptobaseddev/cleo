## What Was Implemented

### T815 — cleo complete strict-mode IVTR enforcement

**Engine function**: `taskCompleteStrict` in `packages/cleo/src/dispatch/engines/task-engine.ts`

- When `lifecycle.mode === 'strict'` AND task has non-null `ivtr_state` AND `ivtr_state.currentPhase !== 'released'` → rejects with `E_IVTR_INCOMPLETE` (exit code 83)
- Error details include `currentPhase`, `failedPhases` list, and `fix` command
- `--force` bypasses check, logs loud owner warning via pino, records `VerificationFailure` in `task.verification.failureLog` (best-effort)
- Returns `ivtrBypassed: true` in response when forced

**New error code**: `E_IVTR_INCOMPLETE: 83` added to `packages/cleo/src/dispatch/engines/_error.ts`

**Domain wiring**: `tasks.ts` `complete` case now calls `taskCompleteStrict` (not `taskComplete`) and passes `force` param

**CLI**: `complete.ts` already had `--force` flag — no CLI change needed

### T817 — cleo show --ivtr-history

**Engine function**: `taskShowIvtrHistory` in `packages/cleo/src/dispatch/engines/task-engine.ts`

- Reads `tasks.ivtr_state` JSON via `getIvtrState`
- Extracts `phaseHistory` array, projects to `IvtrHistoryEntry[]` (renaming `agentIdentity` → `agent`)
- Returns `{ ivtrHistory: [] }` when task has no IVTR state (not an error)

**Output shape**:
```json
{
  "ivtrHistory": [
    {"phase": "implement", "agent": "...", "startedAt": "...", "completedAt": "...", "passed": true, "evidenceRefs": ["sha256..."]},
    {"phase": "validate", "agent": null, "startedAt": "...", "completedAt": null, "passed": null, "evidenceRefs": []}
  ]
}
```

**CLI**: `show.ts` adds `--ivtr-history` option and passes `ivtrHistory` param to dispatch

**Domain wiring**: `tasks.ts` `show` case checks `ivtrHistory` flag before `history` flag
