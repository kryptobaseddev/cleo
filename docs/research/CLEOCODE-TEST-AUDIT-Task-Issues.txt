    Key Problems Found
    
    1. Split twins across directories (T9073, T944)
    
    Same task tested in two different directories with different concerns:
    - t9073-severity-any-role-schema.test.ts in store/tests/ → tests DB CHECK constraint
    - t9073-severity-cross-role.test.ts in tasks/tests/ → tests attestation wiring
    - t944-role-scope-schema.test.ts in store/tests/ → tests DB columns
    - t944-role-scope-wiring.test.ts in tasks/tests/ → tests addTask/findTasks
    
    Each pair should be one canonical file in the right location.
    
    2. Schema parity test pattern is repetitive
    
    All 9 schema tests follow an identical template: create temp dir, run migrations, check PRAGMA table_info, tear down. There's a migration-schema-parity.test.ts that doesn't exist but should — these could be consolidated into ONE parameterized test file with a table-driven approach.
    
    3. Zero explicit test categorization
    
    Of 1,329 test files:
    - 6 .integration.test.ts (0.5%)
    - 2 .e2e.test.ts (0.2%)
    - 0 .unit.test.ts (0%)
    - 63 in tests/integration/ or tests/unit/ folders (CAAMP package only)
    
    No test labeling system exists. Every test is just *.test.ts. There's no way to run "only unit tests" or "only integration tests" in CI.
    
    4. No test hierarchy exists
    
    There's no concept of Unit → Integration → System → Acceptance testing layers. Tests mix concerns freely — a schema parity test (unit) lives next to a full cross-project lifecycle test (integration) with no distinguishing marker.

    Phase 2: Test Taxonomy (1-2 days)
    
    Add naming conventions for explicit test layers:
    
    
    *.unit.test.ts        — Pure unit, no FS/DB/network, fast (<50ms each)
    *.integration.test.ts — Real DB/FS, tmp dirs, no network, run in CI always
    *.e2e.test.ts         — Full CLI invocation, real processes
    *.acceptance.test.ts  — AC verification against task definitions
    
    
    Add vitest project configs for each layer so CI can run:
    - pnpm test:unit → only *.unit.test.ts (fast, pre-commit)
    - pnpm test:integration → *.integration.test.ts (CI gate)
    - pnpm test:e2e → *.e2e.test.ts (release gate)
