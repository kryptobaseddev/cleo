## ConflictPolicy

Policy for resolving conflicts between CLEO and provider state.  - `cleo-wins`: CLEO state takes precedence (default). - `provider-wins`: Provider state takes precedence. - `latest-wins`: Most recently modified value wins. - `report-only`: Report conflicts without applying changes.

## Conduit

The Conduit Protocol interface — high-level agent messaging.  Conduit wraps a Transport adapter, adding messaging semantics.  Implementations: - `ConduitClient` (`@cleocode/core` — wraps any Transport) - `HttpTransport` (HTTP polling to cloud SignalDock API) - `LocalTransport` (napi-rs in-process — embedded SignalDock, future) - `SseTransport` (Server-Sent Events — real-time cloud, future)  Consumers: - `@cleocode/cleo` CLI (`cleo agent watch/poll/send`) - `@cleocode/runtime` (background polling, SSE connections) - CleoOS Electron (embedded SignalDock via LocalTransport) - Agent spawners (deliver task assignments, collect results)

| Property        | Type                                                                                        | Required | Description                                                                            |
| --------------- | ------------------------------------------------------------------------------------------- | -------- | -------------------------------------------------------------------------------------- |
| `send`          | `(to: string, content: string, options?: ConduitSendOptions) => Promise<ConduitSendResult>` | No       | Send a message to an agent or group.                                                   |
| `onMessage`     | `(handler: (message: ConduitMessage) => void) => ConduitUnsubscribe`                        | Yes      | Subscribe to incoming messages. Returns unsubscribe function.                          |
| `poll`          | `(options?: { limit?: number; since?: string; }) => Promise<ConduitMessage[]>`              | No       | One-shot poll for new messages. Returns messages without subscribing.                  |
| `heartbeat`     | `() => Promise<void>`                                                                       | Yes      | Send a heartbeat to indicate this agent is alive.                                      |
| `isOnline`      | `(agentId: string) => Promise<boolean>`                                                     | Yes      | Check if a specific agent is currently online.                                         |
| `listOnline`    | `(() => Promise<string[]>) \| undefined`                                                    | No       | List currently online agents (optional — may not be supported by all implementations). |
| `connect`       | `() => Promise<void>`                                                                       | Yes      | Connect to the messaging backend.                                                      |
| `disconnect`    | `() => Promise<void>`                                                                       | Yes      | Disconnect from the messaging backend.                                                 |
| `getState`      | `() => ConduitState`                                                                        | Yes      | Get the current connection state.                                                      |
| `onStateChange` | `((handler: (change: ConduitStateChange) => void) => ConduitUnsubscribe) \| undefined`      | No       | Subscribe to connection state changes.                                                 |
| `agentId`       | `string`                                                                                    | Yes      | The agent ID this conduit instance is connected as.                                    |
