## Design Decisions

1. **Naming collision avoided**: T763 spec uses `GateResult` but `warp-chain.ts` already exports `GateResult` (LOOM-gate evaluation). Named the acceptance-gate result `AcceptanceGateResult` to prevent re-export collision.

2. **Zod in contracts**: The contracts package previously had zero runtime dependencies. Adding Zod is intentional and required by the task spec. The pure TypeScript types in `acceptance-gate.ts` remain import-free; Zod is isolated to `acceptance-gate-schema.ts`.

3. **Full T763 shape implemented**: The task stub listed a simplified shape. The actual implementation follows T763 §2.1 exactly — all optional fields (`args`, `cwd`, `env`, `advisory`, `timeoutMs`, `startCommand`, `startupDelayMs`, `stderrMatches`, etc.) are present.

4. **`FileAssertion` extended**: T763 defines 8 assertion types including `absent`, `minBytes`, `maxBytes`, and `sha256` beyond the 4 in the task stub. All 8 are implemented.
